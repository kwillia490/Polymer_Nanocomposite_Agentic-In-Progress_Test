Note that I'm providing a an example of polyethylene (PE) for use with REACTER. I've used LUNAR (https://github.com/CMMRLab/LUNAR) to quickly parameterize the PE monomer, pre-reaction, and post-reaction files for PCFF. I built the monomer and post-reaction file with Ketcher (https://lifescience.opensource.epam.com/ketcher/), downloaded these a mol2 files, then built the pre-reaction file from the monomer mol2 file and saved as a pdb file. You can make the start files however you want, just make LUNAR can read the files in (it is fairly versatile).

 Once you have the files, you read them in individually through the atom_typing, selectign PCFF-IFF and True for include_comments_nta. Once the script runs, check the nta file to ensure there were no errors. These will output files with *_typed.data, *_typed.nta, and *_typed.log.lunar.

Then, run the respective *_typed.data and *_typed.nta files through all2imp, selecting the forcefield file from frc_file (these examples use pcff). This will produce files with *_typed_IFF. You will need to make a slight adjust in the Header of the Pre-Reaction and Post-Reaction *_typed_IFF.data files. To each file, which is already added in the provided files, add "; Reduce = [4,4]; BondingIDs = [8,25,8,25]" after Class 2 in the Header. Refer to the LUNAR manual for more detail, but roughly this means we want to reduce the pre-reaction template (*.lmpmol), post-reaction template (*.lmpmol), and mapping file (*_uncommented.txt) to the iniator atoms and atoms 4 steps away from the iniator atoms (which are 8 and 15 in the full pre and post reaction mol2 or pdb files) (You can use Ketcher or other visualizers to see these atom ID numbers). The iniator atoms are listed in the BondingIDs, with the first two being the IDs for the pre-reaction and the last two being the IDs for the post-reaction. Note: These can be different. CHECK! Once you have done this, run the three files through bond_react_merge following the guidance from the LUNAR manual. After this, you'll get several files, but the files you want are akin to the following: PE_PreReaction_typed_IFF_merged.lmpmol, PE_PostReaction_typed_IFF_merged.lmpmol, and pre1-post1_rxn-map_uncommented.txt. 

NOTE: This is where the work with REACTER comes in, requiring you to understand the chemistry and synthesis routes you're studying or trying to model. LUNAR comes with a good handbook, and the creator of REACTER is working on an automated workflow. That said, it is up to you to make sure these required files are good before going to full scale production. I say this because if you look closely at the model for PE monomer I am providing, there is no terminal hydrogen. This means the system may have a charge after polymerization, which presents problems. I'm providing these files so people can play around with the longer polymer workflow as a smaller, "toy model" level. I also encourage you to read through the REACTER pages on LAMMPS, are there are several parameters you should consider trying to optimize based on the monomer you are trying to polymerize. Again, I'm not trying to give a fully finished product, merely provide a starting place better than 0.

Okay, getting off that soapbox, the next step is building a simulation cell, which you can also build with LUNAR's cell builder by reading in the PE_Unit_typed_IFF.data file and selecting the number of monomers you want in the cell (I went with 125 for super small system). Then I adjusted and ran the densify script (PE_IFFDfyS0.IN) followed by the polymerization script (PE_IFFPlyS0.IN) to test and make sure the REACTER files would work to polyermizer the small system. You can check the densified system (PE_IFFDfyS0.dat.gz) and polymerized system (PE_IFFPlyS0.dat.gz) on your visualizer of choice (I've used OVITO for this to use the handy cluster analysis modifier to quickly see that PE went from 125 cluster in the densify step to 3 after polymerization). So, the PE monomer and REACTER files work and should be stable to run through the longer polymer workflow. Again, because the PE monomer is not hydrogen terminated on one end, you'll likely have charge in the system, would could lead to errors in the longer workflow and most definitely is not up to stuff for publication. BUT, you can use this to learn, test, and then go build your actual system on LUNAR and run on a similar polymer workflow!
